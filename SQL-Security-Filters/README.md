# SQL-Based Vulnerability Assessment

## Project Description
In this project, I acted as a security professional investigating potential vulnerabilities within a large organization[cite: 26]. [cite_start]I utilized SQL filters—specifically `AND`, `OR`, and `NOT`—to query the `log_in_attempts` and `employees` tables[cite: 27]. [cite_start]My goal was to isolate suspicious login patterns and audit employee data across various departments to ensure system security and compliance[cite: 28].

## Key SQL Queries
* **After-Hours Failures:** Identified failed attempts after 18:00[cite: 34].
* **Date Investigation:** Reviewed activity from 2022-05-08 and 2022-05-09[cite: 44].
* **Geographic Filtering:** Isolated login attempts from outside of Mexico[cite: 58].
* **Departmental Audits:** Filtered for Marketing, Finance, and Sales staff while excluding IT[cite: 71, 79, 91].
